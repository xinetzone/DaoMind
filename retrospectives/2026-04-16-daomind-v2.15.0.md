## 2. 核心设计

### 架构层次（执行后）

```
DaoUniverse
  ├── DaoUniverseMonitor (v2.8.0)
  │       ├── DaoUniverseClock (v2.9.0)
  │       │       ├── DaoUniverseFeedback (v2.10.0)
  │       │       └── DaoUniverseScheduler (v2.12.0)
  │       │               └── DaoUniverseSkills (v2.13.0)
  │       └── DaoUniverseNexus (v2.14.0)
  └── DaoUniverseAudit (v2.11.0)
          └── DaoUniverseDocs (v2.15.0) ← 知识图谱 × 哲学文档管理
```

### 关键决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 构造参数 | `(audit: DaoUniverseAudit)` | Audit 代表哲学权威，文档发布须经审核 |
| addDoc 双写 | docStore + knowledge graph | 文档即知识节点，加入即自动建图 |
| publishDoc | verifyDoc 门控 | 验证通过方可发布，符合"知常"原则 |
| verifyDoc | 内联验证（无 I/O） | audit.audit() 是昂贵文件扫描，不适合内联 |
| removeDoc | store + graph + published 三同步 | 防止僵尸发布状态 |
| snapshot() | 纯同步 | 不触发 DaoVerificationReporter 文件系统扫描 |

### 新类型

```typescript
DocAuditResult:  { docId, passed, issues, timestamp }
DocsSnapshot:    { timestamp, totalDocs, knowledgeNodes, currentVersion, publishedCount, recentAudit }
```

## 3. DaoApiDocs 适配

**初始预期**：`addApi(api) / getApi(path)`
**实际 API**：`registerApi(Omit<..., 'version'>) / getApi(path, method)`

**修复**：`addApi(api: Omit<DaoApiDescription, 'version'>)` 委托 `_apiDocs.registerApi(api)`；
`getApi(path, method: DaoApiDescription['method'])` 委托 `_apiDocs.getApi(path, method)`

这是典型的"读源再用"原则的验证：规划时须以实际 API 为准。

---

## 4. DaoKnowledgeGraph 适配

**初始预期**：`listNodes() / listEdges()` 方法
**实际 API**：`getStats()` → `{ nodeCount, edgeCount, avgConnections }`

**修复**：`knowledgeStats()` 委托 `_knowledgeGraph.getStats()`，直接取 `nodeCount / edgeCount`

## 5. 关键 Bug（随 v2.15.0 同步修复）

**tsconfig.base.json 缺少路径映射**（根因分析）：

```
症状：12 suites FAIL，ts-jest 报 "Cannot find module '@daomind/anything'"
根因：tsconfig.base.json paths 只覆盖最早集成的 9 个包，
      后续每次集成新包只更新了该包的 tsconfig + daoCollective 的 tsconfig，
      忘记同步更新根 tsconfig.base.json
影响：ts-jest 使用根 tsconfig 做 TypeScript 编译，
      无 paths 则无法解析 workspace 包（无 dist/ 的源码包）
修复：补全 9 条路径：anything/chronos/feedback/skills/pages/spaces/docs/times/collective
经验：每次新增 workspace 包，必须同步检查 tsconfig.base.json paths
```
