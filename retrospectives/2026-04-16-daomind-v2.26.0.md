**版本**：v2.26.0  
**日期**：2026-04-16  
**核心主题**：纯消费者模式 + window 命名陷阱 + noUncheckedIndexedAccess 非空断言

## 一、本版本做了什么

### 新增：DaoUniverseHealthBoard（packages/daoCollective/src/universe-health-board.ts）

v2.26.0 引入 **纯消费者模式** 类——`DaoUniverseHealthBoard`：
不创建任何 `@daomind/*` 子系统实例，只从 `DaoUniverseFacade` 读取数据。

```typescript
const dao   = DaoUniverseFacade.create();
const board = new DaoUniverseHealthBoard(dao);

const entry = board.check();
// { monitorScore, qiNodes, appsRunning, benchRuns, diagCount }

board.trend(); // 'improving' | 'stable' | 'degrading' | 'unknown'
```

**消费者链**（v2.26.0 建立的模式）：
```
DaoUniverseFacade（构建者）
  └──▶ DaoUniverseHealthBoard（消费者）
```

## 二、关键设计决策

### 2.1 check() 数据来源

| 字段 | 来源 |
|------|------|
| `monitorScore` | `facade.snapshot().monitor.systemHealth`（0–100） |
| `qiNodes` | `facade.snapshot().qi.registeredNodes` |
| `appsRunning` | `facade.apps.snapshot().running` |
| `benchRuns` | `facade.snapshot().bench.totalRuns` |
| `diagCount` | `facade.snapshot().diagnostic.totalDiagnoses` |

`facade.snapshot()` 内部已调用 `monitor.feed()`，保证监控数据最新。
`appsRunning` 单独调用 `facade.apps.snapshot()`，因为 `DaoFacadeSnapshot` 不含 `running` 字段（只含 `system.apps.total`，来自不同容器）。

### 2.2 trend() 算法

取最近 `TREND_WINDOW(3)` 条记录，逐步计算相邻 `monitorScore` 差值：
- 所有步骤 > +`TREND_THRESHOLD(0.5)` → `'improving'`
- 所有步骤 < -`TREND_THRESHOLD(0.5)` → `'degrading'`
- 否则 → `'stable'`
- 历史 < 2 条 → `'unknown'`

### 2.3 趋势测试策略

实际 `facade.snapshot()` 返回的 `monitorScore` 因 Agent/Module 状态而变化，
无法精确控制分数序列。

**解决方案**：直接注入 `_history`（TypeScript `as unknown as` 绕过私有访问限制）：
```typescript
(board as unknown as { _history: HealthEntry[] })._history.push(
  { ..., monitorScore: 10 },
  { ..., monitorScore: 20 },
  { ..., monitorScore: 30 },
);
expect(board.trend()).toBe('improving');
```

## 三、调试：两个命名陷阱

### 3.1 window 变量名陷阱

**错误**（初版）：
```typescript
const window = this._history.slice(-TREND_WINDOW);
```
TypeScript 报错：`window[i].monitorScore` 无法访问，因为 `window` 是全局 DOM `Window` 类型。

**修复**：
```typescript
const recent = this._history.slice(-TREND_WINDOW);
```

### 3.2 noUncheckedIndexedAccess 陷阱

**错误**：
```typescript
deltas.push(recent[i].monitorScore - recent[i - 1].monitorScore);
// error TS2532: Object is possibly 'undefined'
```

**根因**：`tsconfig.base.json` 中 `"noUncheckedIndexedAccess": true` 使所有数组索引返回 `T | undefined`。

**修复**：
```typescript
deltas.push(recent[i]!.monitorScore - recent[i - 1]!.monitorScore);
```
