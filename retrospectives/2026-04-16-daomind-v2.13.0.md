### 关键决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 构造参数 | `DaoUniverseScheduler` | 直接取得调度能力与 Clock 引用 |
| 注册中心 | `new DaoSkillRegistry()` | 独立实例，测试隔离，不污染全局 daoSkillRegistry |
| 激活/使用/评分 | 内联实现 | DaoSkillActivator/Scorer/Combiner 硬绑定全局单例，无法注入 |
| 冷却重置 | `scheduler.schedule(→_clearCooling, cooldown)` | 替代 setTimeout，时序统一由 Scheduler 管理 |
| 冷却扫描 | `clock.onTick() → _sweepCooling()` | Clock 心跳触发，状态 cooling→active 无需手动调用 |
| 延迟激活 | `scheduleActivation(id, delayMs)` | 技能可在未来时刻自动激活 |

### SkillEventRecord 类型

```typescript
interface SkillEventRecord {
  readonly skillId: SkillId;
  readonly event: 'activated' | 'deactivated' | 'used' | 'cooled' | 'depleted';
  readonly timestamp: number;
}
```

## 3. 关键实现细节

**独立 DaoSkillRegistry**：所有 daoSkilLs 包中的 Activator/Scorer/Combiner 类都通过 `import { daoSkillRegistry }` 使用全局单例。DaoUniverseSkills 必须绕过它们，直接使用 `new DaoSkillRegistry()` 并内联实现所有逻辑。

**_sweepCooling 与 _clearCooling 分离**：
- `_sweepCooling()`：Clock tick 调用，扫描所有 cooling 技能，比较 `Date.now() - lastUsedAt >= cooldown`
- `_clearCooling(id)`：被 scheduler.schedule() 回调或 _sweepCooling 调用，实际执行状态转换

**combine() 循环依赖检测**：DFS 只检测传入技能集内的环，不检测全局依赖图。
