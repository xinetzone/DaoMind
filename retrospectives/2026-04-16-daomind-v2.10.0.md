# DaoMind v2.10.0 复盘 — 2026-04-16

## 版本：v2.10.0 — DaoUniverseFeedback（daoFeedback × DaoUniverseClock 闭环自调节）

---

## 1. 目标

将 `@daomind/feedback` 的 S 型曲线调节器接入 `DaoUniverseClock.onTick()`，
实现宇宙状态的闭环自调节：health score → signalStrength → RegulationResult。

---

## 2. 核心设计

### 架构层次（执行后）

```
DaoUniverse
  └── DaoUniverseMonitor  (v2.8.0 — 五感引擎，snapshot→MonitorSnapshot)
          └── DaoUniverseClock  (v2.9.0 — daoChronos 时序驱动)
                  └── DaoUniverseFeedback  (v2.10.0 — S型曲线反馈)
```

### `DaoUniverseFeedback` 关键决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 信号转换 | `signalStrength = 100 - health` | 健康越差 → 信号越强 → 调节力越大 |
| 订阅方式 | `clock.onTick((snap) => regulate(snap.systemHealth))` | 复用已建立的 onTick 回调基础设施 |
| 恢复机制 | 每次 `regulate()` 后调用 `regulator.tick()` | 让系统自然衰减，避免永久饱和 |
| 历史滚动 | `MAX_HISTORY = 200`，超出则 `shift()` | 内存可控，无需外部清理 |
| `tick()` | 读取 `monitor.history()` 最后一条 health | 不触发新 capture，避免副作用 |

### 信号转换公式

```
health (0–100) → signalStrength = max(0, 100 - health)
regulator.regulate(signalStrength, windowMs) → RegulationResult {
  outputIntensity ∈ [0,1]  // S型曲线输出
  effectiveSignals          // 实际参与调节的信号数
  droppedSignals            // 被丢弃的过剩信号
  isSaturated               // 是否达到饱和阈值
}
```

## 3. 测试策略

- 6 个**纯同步**构建测试
- 4 个 attach/detach 幂等测试
- 6 个 `regulate()` 输出验证（health=0/100/中间值 / outputIntensity范围 / lastResult更新）
- **4 个 `jest.useFakeTimers()` 自动调节测试**
- 5 个 `history()` 滚动缓冲测试
- 3 个 `tick()` 手动触发测试
- 2 个 regulator 集成测试（getState / 灵敏度对比）
- 2 个 E2E 测试

### 经验

`regulate()` 之后立即调用 `regulator.tick()` 会让 `currentIntensity` 开始衰减，
下一次 `getState()` 返回的 `currentIntensity` 可能略小于 `outputIntensity`，测试时注意区分两者。
