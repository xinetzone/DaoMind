#### 架构设计

```
.app (100dvh, flex column)
├── .app-nav (52px TopNav)
│   ├── [DaoLogo] DaoMind · Modulux（→ navigateChat()）
│   ├── .app-nav-links: 指南 / API / 示例 / FAQ（→ navigateDocs(defaultPath)）

└── .app-content (flex: 1, overflow: hidden)
    ├── #chat → ChatPage
    └── #docs/<path> → DocsPage
        ├── DocSidebar（240px）
        └── .docs-content → MystRenderer
```

路由方案选择：**哈希路由**（`window.location.hash` + `hashchange` 事件），原因：
- 无需 react-router 依赖
- SPA 在任意路径都可访问（无 404 问题）
- 简单、可控、zero-dependency

#### 渲染管线

```
.md string
  -> stripFrontmatter()         去除 YAML/TOML front-matter
  -> preprocessDirectives()     MyST 指令转 HTML div（admonition）
  -> ReactMarkdown (v10)
       remarkPlugins: [remarkGfm]       GFM 表格、删除线等
       rehypePlugins: [rehypeHighlight]  代码语法高亮
       components: pre/code/table/blockquote/a 自定义渲染
  -> React DOM
```

`preprocessDirectives` 将 `:::info Title` 格式的指令转换为 `<div class="admonition admonition-info">` HTML，支持类型：note / info / tip / warning / caution / danger / important。

#### 文档内容加载策略

使用 Vite `import.meta.glob` + `eager: true` 在构建时静态打包所有 `.md` 文件：

```typescript
const allDocs = import.meta.glob('/docs/site/**/*.md', {
  query: '?raw', import: 'default', eager: true
}) as Record<string, string>;
```

- 优点：零运行时网络请求，无服务器依赖，构建后文档内容内嵌 bundle
- 代价：bundle 体积增加（17 个 .md 文件约 40KB 未压缩）

路径映射：`pathToGlobKey('/guide/getting-started')` → `/docs/site/guide/getting-started.md`

### v2.29.1：AI 响应时间优化（perf）

#### 触发
用户反馈：「响应时间太长」

#### 根因分析

首 token 延迟（TTFB）过长，主要来自两处：
1. **系统提示词过长**：原版约 280 tokens（多行详细知识库），GLM 5 需先处理全部输入才开始生成
2. **`max_tokens: 2048` 偏大**：最大生成时长偏高

#### 优化效果

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| 系统提示词 | ~280 tokens | ~80 tokens | TTFB ↓ 约 60% |
| max_tokens | 2048 | 1200 | 最大生成时间 ↓ 约 40% |
| Edge Function 代码 | 97 行 | 60 行 | Cold start ↓ |

精简后保留的核心知识：帛书版本结构（德经在前）/ 关键异文「中气以为和」/ 三才（天地人）× 中气调和 / 四十二章宇宙生成论（道→一→二→三→万物）

## 遇到的问题与解决

### 问题 1：pnpm add 在 monorepo 中失败

**错误**：`pnpm add react-markdown remark-gfm ...` 报 no package.json found

**原因**：monorepo workspace root，不指定 `-w` 时拒绝安装到 root

**解决**：所有依赖安装加 `-w` flag：`pnpm add ... -w`

---

### 问题 2：write_file 因转义字符拒绝写入

**错误**：MystRenderer.tsx 写入失败，JSDoc 注释中含字面量反斜杠-n 序列（如代码示例中的多行指令展示）

**解决**：删除含反斜杠-n 字面量的 JSDoc 注释行，直接写代码不加示例注释

---

### 问题 3：Lint warning — cleanup 函数缺少返回类型

**位置**：`src/App.tsx` line 36

**错误**：cleanup 函数 Missing return type on function expression

**解决**：`return (): void => window.removeEventListener('hashchange', handler)`
