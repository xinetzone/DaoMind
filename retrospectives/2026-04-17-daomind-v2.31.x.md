## 二、关键决策：移除文档模块

### 背景
v2.29.0 集成了基于 MyST Markdown + import.meta.glob 的 Web SPA 文档站，但实际使用体验
未达到 VitePress 参考站的水准。用户明确表示失望，希望聚焦 AI 问答核心功能。

### 移除范围

**删除文件（4 个）：**
- `src/pages/DocsPage.tsx` — 文档页面主组件
- `src/components/DocSidebar.tsx` — 文档侧边栏
- `src/components/MystRenderer.tsx` — MyST/rehype Markdown 渲染器
- `src/data/navigation.ts` — 文档导航数据（73 个路由定义）

**移除 npm 包（3 个）：**
- `rehype-raw` — HTML 字符串解析（MyST admonition 渲染需要）
- `rehype-highlight` — 代码语法高亮（依赖 highlight.js）
- `remark-directive` — MyST `:::note` 指令解析

**CSS 清理（-945 行）：**
| 区间 | 内容 | 操作 |
|------|------|------|
| 145-707 | 旧版 Landing Page（Navbar/Hero/Sections 等遗留样式） | 删除 |
| 1327-1708 | Docs Layout + MyST 渲染 + Admonition | 删除 |
| 708-1326 | Chat Layout + App Shell | 保留 |
| 1709-1823 | chat-md（AI 响应 Markdown 样式） | 保留 |

### 量化影响
```
index.css  : 1823 行 → 878 行  （-51.7%）
App.tsx    : 85 行  → 22 行   （-74%）
build      : 2244 modules → 1976 modules（-12%，构建 295ms）
```

## 三、Prettier 格式化复盘（v2.30.5）

### 根本原因
`.prettierrc` 存在于仓库（`semi: false, singleQuote: true, tabWidth: 2`），但 `prettier`
从未安装为 devDependency，导致历史提交各自为政。

### 修复规模
- `src/**/*.{ts,tsx,css}` 15 个文件
- `supabase/functions/**/*.ts` 2 个文件
- `docs/site/**/*.md` 21 个文件（代码块去分号 + 表格对齐）
- 共 38 个文件，2584 insertions / 2091 deletions

### 预防措施
新增 `pnpm format` 脚本，今后在每次批量修改后执行。

**data-lang 语言标签（v2.30.4）：**
```tsx
// pre 组件提取子 code 的 className
const langMatch = className.match(/language-(\w+)/)
const lang = langMatch ? langMatch[1] : ''
return <div className="doc-pre" data-lang={lang || undefined}>{children}</div>
```
```css
.doc-pre[data-lang]::after {
  content: attr(data-lang);  /* CSS 读取 data 属性显示标签 */
  position: absolute; top: 0.65rem; right: 1rem;
  color: rgba(255,255,255,0.35);
}
```

## 六、遗留技术债

1. **chat-md 样式孤立** — `src/index.css` 中 `.doc-*` 前缀类已无对应组件，
   但 `chat-md` 样式使用 `hsl(var(--primary))` 等 CSS 变量，需保留。
2. **`app-nav-chat-btn` / `app-nav-links` 样式残留** — v2.31.0 中 App.tsx 已不使用
   这些类，但 CSS 中仍存在，可在下次清理时一并删除。
3. **Worker 退出警告** — Jest 测试结束时有 `force exited` 警告，提示存在未清理的
   异步句柄（`useAIChat.ts` 中的 `fetchEventSource` 或 `setTimeout`），
   可用 `--detectOpenHandles` 追踪。
