# ZenFrame & Modulux 品牌命名规范

## 1. 品牌名称

### 1.1 主品牌名称
- **ZenFrame**：整体框架品牌
- **Modulux**：模块化组件库品牌

### 1.2 品牌名称使用规则
- 品牌名称应始终保持完整拼写
- 首字母大写，其余字母小写
- 避免使用缩写或简称
- 在正式文档和通信中使用完整品牌名称

### 2.3 命名规则

**产品命名规则**：
- 使用品牌名称作为前缀（ZenFrame 或 Modulux）
- 产品名称应简洁明了，反映产品的核心功能
- 使用 PascalCase 命名法（首字母大写）
- 避免使用技术术语和行话
- 确保名称在行业内具有独特性

**示例**：
- ✅ ZenFrame Core
- ✅ Modulux UI
- ❌ ZenFrame-core
- ❌ ModuluxUI

## 3. 版本命名规范

### 3.1 版本号格式

**主版本号.次版本号.补丁号**
- **主版本号**：重大变更，不兼容的 API 更改
- **次版本号**：新功能，向后兼容
- **补丁号**：bug 修复，向后兼容

**示例**：
- 1.0.0
- 1.1.0
- 1.1.1

### 3.2 版本类型

**稳定版**：
- 格式：x.y.z
- 示例：1.0.0

**预发布版**：
- 格式：x.y.z-beta.n
- 示例：1.0.0-beta.1

**开发版**：
- 格式：x.y.z-dev.n
- 示例：1.0.0-dev.1

### 3.3 版本命名规则

- 版本号应按顺序递增
- 主版本号从 1 开始
- 次版本号和补丁号从 0 开始
- 预发布版和开发版应在版本号后添加相应的标识
- 版本号应保持一致的格式

### 3.4 版本发布流程

1. **开发版**：用于内部开发和测试
2. **预发布版**：用于公开测试和反馈
3. **稳定版**：正式发布的版本

## 4. 包命名规范

### 4.1 NPM 包命名

**ZenFrame 包**：
- 格式：@zenframe/{包名}
- 示例：@zenframe/core

**Modulux 包**：
- 格式：@modulux/{包名}
- 示例：@modulux/ui

### 4.2 包命名规则

- 使用小写字母和连字符
- 包名应简洁明了，反映包的核心功能
- 避免使用技术术语和行话
- 确保包名在 NPM  registry 中具有独特性

## 5. 命名空间规范

### 5.1 代码命名空间

**ZenFrame 命名空间**：
- 格式：ZenFrame
- 示例：ZenFrame.Core

**Modulux 命名空间**：
- 格式：Modulux
- 示例：Modulux.UI

### 5.2 命名空间使用规则

- 使用 PascalCase 命名法
- 命名空间应反映代码的组织结构
- 避免使用过长或复杂的命名空间
- 确保命名空间在代码库中具有唯一性

## 6. 文档命名规范

### 6.1 文档命名

- 使用小写字母和连字符
- 文档名称应简洁明了，反映文档的内容
- 避免使用技术术语和行话
- 确保文档名称在文档系统中具有唯一性

**示例**：
- ✅ zenframe-core-getting-started.md
- ✅ modulux-ui-components.md
- ❌ ZenFrameCoreGettingStarted.md
- ❌ ModuluxUICOmponents.md

### 6.2 文档结构

- 按照产品和功能组织文档
- 使用一致的文档结构和格式
- 确保文档内容与产品版本保持同步
- 提供清晰的文档导航和索引

## 7. 品牌命名审核

### 7.1 命名审核流程

1. **提案**：提交新的产品或功能名称
2. **审核**：审核名称是否符合命名规范
3. **批准**：批准符合规范的名称
4. **实施**：在产品和文档中使用批准的名称

### 7.2 命名审核标准

- 是否符合品牌定位和价值主张
- 是否符合命名规范
- 是否在行业内具有独特性
- 是否易于发音和记忆
- 是否适合品牌推广和市场传播

## 8. 命名变更管理

### 8.1 命名变更流程

1. **评估**：评估命名变更的必要性和影响
2. **提案**：提交命名变更提案
3. **审核**：审核命名变更提案
4. **批准**：批准命名变更提案
5. **实施**：实施命名变更
6. **沟通**：向用户和合作伙伴沟通命名变更

### 8.2 命名变更注意事项

- 考虑向后兼容性
- 提供迁移指南
- 确保所有相关文档和资源都得到更新
- 避免频繁的命名变更

## 9. 最佳实践

- **简洁性**：使用简洁明了的名称
- **一致性**：保持命名的一致性
- **独特性**：确保名称在行业内具有独特性
- **可扩展性**：考虑未来的扩展和发展
- **品牌一致性**：确保名称与品牌定位一致

- **避免使用**：
  - 技术术语和行话
  - 过长或复杂的名称
  - 与其他品牌冲突的名称
  - 可能引起混淆的名称
