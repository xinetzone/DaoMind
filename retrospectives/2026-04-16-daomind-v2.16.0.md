### 关键决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 构造参数 | `(nexus: DaoUniverseNexus)` | Nexus 提供服务注册/路由/健康检查能力 |
| createSpace 双写 | namespace + nexus.register | 空间即服务，路由层可直接寻址 |
| endpoint 格式 | `space://${spaceId}` | 语义清晰，避免与 HTTP 路由混淆 |
| removeSpace | namespace.removeSpace + nexus.deregister | 有子空间时抛出（委托 DaoNamespaceManager 保护） |
| dispatch 路由原理 | `path.split('/')[0]` = serviceName | dispatch 以路径首段为服务名，createSpace 的 name 需匹配 |
| routeSpace | 可选路由规则 | 无规则时 discover 动态路由；有规则时 router 优先 |
| _countAllSpaces() | DFS 递归 | DaoNamespaceManager 无 size API，需通过 getChildren + getRootSpaces 遍历 |

### 新类型

```typescript
SpacesSnapshot: { timestamp, totalSpaces, rootCount, nexusServiceCount }
```

## 3. DaoRouteRule 适配

**初始预期**：`{ pattern, target, priority }` 三字段
**实际 API**：`{ pattern, target, weight, priority }` 四字段（weight 必填）

**修复**：`routeSpace()` 中补 `weight: 1`

这是典型的"读 types.ts 再动手"原则验证：路由规则字段须与 DaoRouteRule 接口精确匹配。

---

## 4. dispatch 路由路径格式

**关键发现**：`dispatch({ path: '/tasks/42' })` → `svcName = ''`（空字符串，因 split('/')[0] = ''）
**正确格式**：`dispatch({ path: 'tasks/42' })` → `svcName = 'tasks'`

因此测试中所有 dispatch 调用均使用**无 leading slash** 的路径格式。
